# БЫСТРЫЙ СТАРТ Parser Bot 🚀

## За 5 минут до первого парсинга!

### Шаг 1: Подготовка (2 мин)

```bash
# 1. Получите токен бота
# Откройте Telegram и напишите @BotFather
# Команда: /newbot
# Следуйте инструкциям и получите токен

# 2. Скопируйте .env
cp .env.example .env

# 3. Отредактируйте .env и вставьте токен
# Откройте .env и замените:
# TELEGRAM_BOT_TOKEN=your_bot_token_here
# на ваш настоящий токен
```

### Шаг 2: Установка зависимостей (2 мин)

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

**macOS/Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Шаг 3: Запуск (1 мин)

```bash
python bot.py
```

Вы должны увидеть:
```
✅ Бот запущен! Нажмите Ctrl+C для остановки.
```

### Шаг 4: Тестирование

1. Откройте Telegram
2. Найдите бота по имени
3. Нажмите /start
4. Попробуйте выполнить поиск

## Первый поиск - Пошаговое руководство

### Пример: Поиск кафе в Алматы на 2ГИС

1. **Запустите бота** и нажмите `/start`
2. **Выберите** "🔍 Новый поиск"
3. **Выберите источник** → "2ГИС"
4. **Выберите город** → "Алматы"
5. **Введите запрос** → "кафе" или "электроника"
6. **Ждите результатов** → Бот будет искать 10-30 секунд
7. **Экспортируйте** → Нажмите "⬇️ Экспортировать" и выберите формат

## Часто задаваемые вопросы

### Q: Бот не отвечает
**A:** 
1. Проверьте токен в .env
2. Убедитесь что бот запущен (должна быть строка "✅ Бот запущен!")
3. Перезагрузите Telegram

### Q: Парсинг занимает долго
**A:** 
1. Это нормально - первый запрос может занять 10-30 секунд
2. Последующие запросы могут быть быстрее благодаря кешу
3. Если очень долго - попробуйте использовать VPN

### Q: Как увеличить количество результатов?
**A:** 
1. Отредактируйте `bot.py` строку с `limit=50`
2. Измените на нужное число (например, `limit=100`)
3. Перезагрузите бота

### Q: Где находятся экспортированные файлы?
**A:** 
1. В папке `data/exports/`
2. Файлы скачиваются также через Telegram

### Q: Как добавить новый источник парсинга?
**A:** 
1. Создайте новый файл в `parsers/mynewparser.py`
2. Реализуйте класс с методом `search()`
3. Добавьте в `parser_manager.py`

## Деплой на сервер 🚀

### Вариант 1: VPS (Linux)

```bash
# 1. SSH на сервер
ssh user@your-vps.com

# 2. Клонируем проект
git clone <repo>
cd parser

# 3. Создаём виртуальное окружение
python3 -m venv venv
source venv/bin/activate

# 4. Устанавливаем зависимости
pip install -r requirements.txt

# 5. Создаём .env с токеном
cp .env.example .env
nano .env  # Отредактируйте токен

# 6. Запускаем в фоне (tmux)
tmux new-session -d -s parser python bot.py

# Проверить статус:
tmux attach-session -t parser
```

### Вариант 2: Docker (рекомендуется)

```bash
# 1. На сервере
git clone <repo>
cd parser

# 2. Создаём .env
cp .env.example .env
# Отредактируйте .env

# 3. Запускаем
docker-compose up -d

# Проверить логи:
docker-compose logs -f
```
